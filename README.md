# Лабораторна робота №4 та №5 - Консольний застосунок з використанням Spring Boot і JPA для опрацювання декількох таблиць БД з реалізацією CRUD-операцій та зв’язків між таблицями

- Lab4Application (main)
    - Клас є точкою входу застосунку і містить тільки метод main.
    - Позначений анотацією @SpringBootApplication, що вмикає автоконфігурацію Spring Boot та сканування компонентів.
    - Використовується JVM для запуску, а далі Spring створює всі біни (сервіси, репозиторії, CliView тощо).

- Project (model)
    - Клас представляє сутність проєкту, яка мапиться на таблицю Projects у базі даних.
    - Містить поля id, url і type з JPA-анотаціями та Lombok @Getter/@Setter для генерації геттерів/сетерів.
    - Використовується Hibernate/JPA при збереженні, читанні та оновленні проєктів через ProjectRepository і ProjectService.

- Type (model)
    - Клас представляє сутність типу проєкту, що мапиться на таблицю Types.
    - Містить поля id та name з JPA-анотаціями і Lombok @Getter/@Setter для роботи з ними.
    - Використовується як ціль зв’язку Many-to-One у Project та у сервісах/репозиторіях для керування типами.

- ProjectRepository (interface)
    - Інтерфейс репозиторію для сутності Project, який успадковується від JpaRepository.
    - Надає стандартні CRUD-операції та додатковий метод findByUrl для пошуку проєкту за унікальним URL.
    - Використовується в ProjectService для отримання списку проєктів, створення, оновлення, видалення та перевірки дублювання URL.

- TypeRepository (interface)
    - Інтерфейс репозиторію для сутності Type, який успадковується від JpaRepository.
    - Окрім CRUD, містить метод findByName для пошуку типу за унікальною назвою.
    - Використовується в TypeService для CRUD-операцій над типами та в ProjectService для пошуку типу по імені при створенні/оновленні проєкту.

- ProjectService (service)
    - Клас інкапсулює бізнес-логіку CRUD для проєктів та їх зв’язку з типами.
    - Використовує ProjectRepository для читання/запису таблиці Projects і перевірки унікальності URL, а також TypeRepository для пошуку типів по назві.
    - Методи createProject, updateProject, deleteProject повертають текстові результати операцій і викликаються консольною в’юшкою CliView.

- TypeService (service)
    - Клас інкапсулює бізнес-логіку CRUD для типів проєктів.
    - Використовує TypeRepository для створення, оновлення, видалення та отримання списку всіх типів.
    - Методи createType, updateType, deleteType повертають текстові повідомлення, які виводяться користувачу з CliView при роботі з таблицею Типи.

- CliView (view)
    - Клас відповідає за консольний інтерфейс: показує головне меню, читає введення користувача та делегує дії сервісам.
    - Використовує TypeService і ProjectService для виконання CRUD-операцій над таблицями Types і Projects, реалізуючи логіку C/U/D та перевірку порожності таблиць.
    - Має окремі методи handleTypes, handleProjects, handleTypesProjects і parseId, де третій метод лише відображає зв’язки Project–Type без можливості прямої зміни третьої таблиці.
